_ Footer :

To finalize the application of the Traditional Box Model, another CSS property has to be applied to the footer element. This property returns the document to its normal flow and allows us to position the footer below the last element instead of next to it:

text-align: center;
padding: 20px;
border-top:2px solid #999999; }
Listing 2-34. Giving styles to footer and recovering the normal flow of the document.

The rule in Listing 2-34 declares a border of 2 pixels at the top of the footer, a padding of 20 pixels, and centers the text within the element. Itself restores the normal flow of the document with the clear property. This property simply restores the normal conditions of the area occupied by the element, not allowing it to be positioned adjacent to a floating box. The value usually used is both, which means that both sides of the element will be restored and the element will follow the normal flow (this element is no longer floating like the previous ones).

This, for a block element, means which will be positioned below the last element, on a new line. The clear property also pushes elements vertically, making the floating boxes occupy a real area on the screen. Without this property, the browser renders the document in screen as if the floating elements did not exist and the boxes overlap. Figure 2-2. Visual representation of the model traditional box.

When we have boxes positioned next to each other in the Traditional Box Model we always need to create an element with the style clear: both so that you can continue to add other boxes below in a natural way. Figure 2-2 shows a visual representation of this model with the basic styles to achieve the correct layout on the screen. The left and right values of the float property do not mean that the boxes must necessarily be positioned on the left or right side of the window. What the values ​​do is float that side of the element, breaking the normal flow of the document.

If the value is left, by For example, the browser will try to position the element on the left side in the available space. If space is available then of another element, this new element will be placed to its right, because its left side was set to float. The element it floats to the left until it finds something blocking it, like another element or the edge of its parent element. This is important when we want to create several columns on the screen. In this case each column will have the value left in the property float to ensure that each column will be continuous with the other in the correct order. In this way, each column will float to the left until it is blocked by another column or the edge of the parent element.

- finishing touches :
The only thing left for us to do is work on the design of the content. For this, we only need to configure the few remaining HTML5 elements:

article { background: #FFFBCC; border: 1px solid #999999; padding: 20px; margin-bottom: 15px; }
article footer { text-align: right; }
time { color: #999999; }
figcaption { font: italic 14px verdana, sans-serif; }
Listing 2-35. Adding the finishing touches to our basic design.

The first rule in Listing 2-35 references all the articl> elements and gives them some basic styles (background color, a solid 1 pixel border, inner margin and bottom margin). The 15 pixel bottom margin is meant to separate one article element from the next. vertically. Each article element also has a footer element that displays the number of comments. received. To reference a footer element within an article element, we use the article selector footer which means “each footer within an article will be affected by the following styles”. This technique reference was applied here to right-align the text within the footer elements of each article. At the end of the code in Listing 2-35 we change the color of each time element and differentiate the description of the image (inserted with the figcaption element) from the rest of the text using a different font.
