_ Text-shadow :

Now that you know all about shadows, you're probably thinking of generating one for each element in your document. The box-shadow property was designed especially to be applied to boxes. If you try to apply this effect to an element span, for example, the invisible box occupied by this element on the screen will have a shadow, but not the content of the element. To create shadows for irregular shapes like text, there is a special property called text-shadow:


Listing 3-9. Generating a shadow for the title.

The values ​​for text-shadow are similar to those used for box-shadow. We can declare the shadow color, the distance horizontal and vertical of the shadow with respect to the object and the radius of blur. In Listing 3-9 a blue shade was applied to the title of our template with a distance of 3 pixels and a blur radius of 5.

- @font-face :
Getting a text with shadow is actually a very nice design trick, impossible to achieve with previous methods, but rather than changing the text itself it only provides a three dimensional effect. A shadow, in this case, is like paint an old car, in the end it will be the same car. In this case, it will be the same font. The problem with fonts or typefaces is as old as the web.

Regular web users often have a limited number of fonts installed on their computers, usually these fonts are different from user to user, and most Most of the time many users will have fonts that others will not. For years, websites could only use a limited trusted fonts group (a basic group that almost all users have installed) and thus present the information on screen.

The @font-face property allows designers to provide a file containing a font specific to display your texts on the page. Now we can include any font we need just by providing the proper file:


Listing 3-10. New font for the title.

Do it yourself :
Download the font.ttf file from our website or use one you already have and copy it to the same directory (folder) as your CSS file. To download the file.

IMPORTANT :
The file containing the font must be on the same domain as the web page (or on the same computer, in this case). This is a restriction of some browsers like Firefox for example.

The @font-face property needs at least two styles to declare the font and load the file. The style built with the font-family property specifies the name we want to give to this particular font, and the src property indicates the URL of the file with the code corresponding to that source. In Listing 3-10, the name MyNewSource was assigned to our new typeface and font.ttf file was indicated as the file corresponding to this font.

Once the font is loaded, we can start using it in any element of the document by simply typing its name (MyNewSource). In the font style in the rule in Listing 3-10, we specify that the title will be displayed with the new font or verdana and sans-serif alternatives in case the embedded font is not loaded properly.
