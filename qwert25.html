_ Playing video with HTML5:

One of the most mentioned features of HTML5 was the ability to render video. The enthusiasm had nothing to do with the new tools provided by HTML5 for this purpose, but rather with the fact that since the videos became an essential piece of the Internet, everyone expected native support from browsers. It was like everyone knew the importance of videos except those in charge of developing the technologies for the web.

But now that we have native support for videos and even a standard that will allow us to create cross-browser compatible video processing applications, we can understand that the situation was much more complicated than we had imagined. From encoders to resource consumption, the reasons to not implement video natively in browsers were much more complex than the code required to do so. A Despite these complications, HTML5 finally introduced an element to insert and play video in an HTML document. The video element uses opening and closing tags and only a few parameters to achieve its function. The syntax is extremely simple and only the src attribute is required:

First we must provide at least two different files with different video formats: OGG and MP4. This is because although the video element and its attributes are standard, there is no standard video format. First, some browsers support a video encoder that others don't, and second, the encoder used in the MP4 format (the only supported by major browsers such as Safari and Internet Explorer) is under a commercial license. The OGG Formats and MP4 are containers for video and audio. OGG contains Theora video and Vorbis audio encoders, and those available for the MP4 container are H.264 for video and AAC for audio. Right now OGG is recognized by Firefox, Google Chrome and Opera, while MP4 works on Safari, Internet Explorer and also Google Chrome.

- The video element :
Let's try to ignore these complications for a moment and enjoy the simplicity of the video element. This item offers several attributes to set its behavior and configuration. The width and height attributes, just like in other HTML elements already known, declare the dimensions for the player element or window. The size of the video will be automatically adjusted to enter within these values, but they were not considered to resize the video but to limit the area occupied by the same to maintain consistency in the design. The src attribute specifies the source of the video. - Attributes for video :
We included an attribute on the video tag in Listings 5-1 and 5-2 that probably caught your attention. The controls attribute is one of several attributes available for this element. This one, in particular, shows video controls provided by the default browser. When the attribute is present each browser will activate its own interface, allowing the user to start playing the video, pause it or skip forward. a specific box, among other functions. Along with controls, we can also use the following:

- Autoplay :
When this attribute is present, the browser will automatically start playing the video as soon as it can.

- loop :
If this attribute is specified, the browser will start playing the video again when it reaches the end.

- poster :
This attribute is used to provide an image that will be displayed while we wait for the video to start playing.

- preload :
This attribute can receive three different values: none, metadata or auto. The first indicates that the video is not should be cached, usually for the purpose of minimizing unnecessary traffic. The second value, metadata, will recommend to the browser that tries to capture information about the source (for example, dimensions, duration, first frame, etcâ€¦). The third value, auto, is the default value that will prompt the browser to download the file as soon as possible.
