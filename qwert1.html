Each part of the document is differentiated, declared and determined by specific tags. In this part of the chapter we are going to see how to build the global structure of an HTML document and the new semantic elements built into HTML5.

-Using the element:
The lang attribute in the opening html tag is the only attribute we need to specify in HTML5. This attribute defines the human language of the content of the document that we are creating, in this case it is Spanish. Basics: HTML uses a markup language to build web pages. These HTML tags are words key and attributes surrounded by greater than and less than signs (for example, html lang="en"). In this case, html is the keyword and lang is the attribute with the value is. Most HTML tags are used in pairs. an opening tag and a closing tag, and the content is declared between them. In our example, html lang="en" indicates the beginning of the HTML code and /html indicates the end. Compare opening tags and close and you will see that the closing is distinguished by a backslash before the keyword (for example, /html). HTML5 is extremely flexible in terms of structure and elements used to build it. The html element can be included without any attributes or even completely ignored. The embedded HTML code between tags has to be divided between two main sections. As in previous versions of HTML, the first section is the header and the second is the body. The next step, therefore, will be to create these two sections in the code using the elements already known. The head element comes first, of course, and like the rest of the structural elements has an opening tag and a closing tag: !DOCTYPE html html lang="en" head /head /html

- Using the head element:.
The tag has not changed from previous versions and its purpose remains exactly the same. Inside the head tags we will define the title of our web page, we will declare the set of corresponding characters, we will provide general information about the document and incorporate the external files with styles, Javascript codes or even images necessary to generate the page on the screen. Except for the title and some icons, the rest of the information embedded in the document between these tags is invisible to the user. body The next big The section that is the main part of the organization of an HTML document is the body. The body represents the visible part of every document and is specified between body tags. These labels have also not changed relative to previous versions of HTML:

-Using the title tag:
Basic concepts: The text between the title tags is the title of the document we are creating. Normally this text is displayed in the top bar of the browser window. link Another important element that goes inside the document header is link. This element is used to incorporate styles, Javascript codes, images or icons from external files. One of the most common uses for link is to embed files with CSS styles:

-Using the link element:
In HTML5 we no longer need to specify what kind of styles we are inserting, so the type attribute was removed. We only need two attributes to incorporate our style file: rel and href. The rel attribute means "relationship" and is about the relationship between the document and the file that we are incorporating through href. In this case, the rel attribute has the stylesheet value that tells the browser that the mystyles.css file is a CSS file with required styles to present the page on the screen (in the next chapter we will study how to use CSS styles). Basics: A styles file is a group of formatting rules that will help change the appearance of our website (for example, the size and color of the text).
Without these rules, the text and any other HTML elements would be displayed on the screen using the standard styles provided by the browser. Styles are simple rules that typically require only a few lines of code. and can be declared in the same document. As we will see later, it is not strictly necessary get this information from external files but it is a best practice. Load CSS rules from an external document (another file) will allow us to organize the document main, increase the loading speed and take advantage of the new features of HTML5.
