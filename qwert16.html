_ transform and transition :

HTML elements, when created, are like solid, immovable blocks. They can be moved using Javascript code or by taking advantage of popular libraries like jQuery (www.jquery.com), for example, but there was no standard procedure for this purpose until CSS3 introduced the transform and transition properties. Now we no longer have to think about how to do it. Instead, we just have to know how adjust a few parameters and our website can be as flexible and dynamic as we imagine it to be. The transform property can operate four basic transformations on an element: scale, rotate, skew, and translate. Let's see how do they work:

- Transform: scale :


Listing 3-19. Changing the scale of the header box.

In the example in Listing 3-19 we start with the basic styles used for the header generated in Listing 3-2 and apply transform by doubling the scale of the element. The scale function receives two parameters: the X value for the horizontal scale and the value And for the vertical scale. If only one value is provided the same value is applied to both parameters. Integers and decimals can be declared for scale. This scale is calculated by means of a matrix. Values ​​between 0 and 1 will shrink the element, a value Values ​​of 1 will keep the original proportions, and values ​​greater than 1 will increase the dimensions of the element incrementally. The second value also maintains the original aspect ratio, but flips the element vertically to produce the mirror effect. There are also two other functions similar to scale but restricted to the horizontal or vertical dimension: scaleX and scaleY. These functions, of course, use only one parameter.

- Transform: translate :
Similar to the old top and left properties, the translate function moves or scrolls the element on the screen to a new position.


Listing 3-23. Moving the header box to the right.

The translate function considers the screen as a grid of pixels, with the element's original position used as a reference point. The top left corner of the element is position 0,0, so negative values ​​will move the object to the left or up from the original position, and positive values ​​will make it to the right or down. In Listing 3-23, we moved the header box to the right about 100 pixels from its original position. two values can be declared in this function if we want to move the element horizontally and vertically, or we can use functions separate calls translateX and translateY.
