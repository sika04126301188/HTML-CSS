_ Border-radius :

For many years designers have suffered trying to achieve the effect of rounded corners in the boxes of their web pages. The process it was almost always frustrating and exhausting. Everyone has suffered from it. If you watch any video presentation of the new built-in features in HTML5, whenever someone talks about CSS3 properties that make it possible to easily generate rounded corners, the audience goes crazy. Rounded corners were the kind of things that made us think: “it should be easy to do”. However it never was. This is the reason why, among all the new possibilities and incredible properties built into CSS3, the one we'll explore first is border-radius:


Listing 3-3. Generating rounded corners.

The border-radius property is currently experimental so we must use the prefixes –moz- and -webkit- so that work in browsers based on Gecko and WebKit engines, such as Firefox, Safari and Google Chrome (the prefixes were studied and applied in Chapter 2). If all the corners have the same curvature we can use a single value. However, as happens with the margin and padding properties, we can also declare a different value for each one:


Listing 3-4. Different values ​​for each corner.

As you can see in Listing 3-4, the four values ​​assigned to the border-radius property represent different locations. Moving around the box clockwise, the values ​​will be applied in the following order: top left corner, upper right corner, lower right corner and lower left corner. Values ​​are always given in the direction of the clockwise, starting at the top left corner. As with margin or padding, border-radius can also work only with two values. The first value will be assigned to the first and third corners (upper left, lower right), and the second value to the second and fourth corners (top right, bottom left). We can also shape the corners by declaring a second group of values ​​separated by a slash. The values ​​to the left of the bar will represent the horizontal radius while that the values ​​to the right represent the vertical radius. Combining these values ​​generates an ellipsis:

Box-shadow Now that finally we have the possibility of generating nice corners for our boxes we can take a risk with something else. Other very good effect, which had been extremely difficult to achieve until now, is shadows. For years designers have combined images, elements and some CSS properties to generate shadows. Thanks to CSS3 and the new box-shadow property we can apply shadows to our boxes with just a simple line of code:


Listing 3-6. Applying shadow to our box.

The box-shadow property needs at least three values. The first, which you can see in the rule in Listing 3-6, it is the color. This value was constructed here using the rgb() function and decimal numbers, but we can write it in hexadecimal numbers as well, as we did previously for other parameters in this book. the next two values, expressed in pixels, set the offset of the shadow.

This offset can be positive or negative. negative. The values ​​indicate, respectively, the horizontal and vertical distance from the shadow to the element. Negative values ​​will position the shadow to the left and above the element, while positive values will create the shadow to the right and below the element. Values ​​of 0 or null will position the shadow exactly behind the element, allowing the possibility of creating a blurred effect all around it.
