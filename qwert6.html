_ Traditional box model :

It all started with boards. The tables were the elements that unintentionally became the ideal tool used by developers to create and organize content boxes in the screen. This can be considered the first box model on the web. Boxes were created by expanding cells and combining rows of cells, columns of cells and entire tables, one on top of another or even nested. when the sites websites grew and became more and more complex this practice began to present serious problems related to the size and maintenance of the code needed to create them. These initial problems necessitated what is now we see as a natural practice: the division between structure and presentation.

Using div tags and CSS styles was possible to override the tables function and effectively separate the HTML structure from the presentation. With div elements and CSS we can create boxes on the screen, position these boxes to one side or the other and give them a specific size, color or border among other characteristics. CSS provides specific properties that allow us to organize the boxes according to our wishes. These properties are powerful enough as to create a box model that became what we know today as a Box Model Traditional.

Some deficiencies in this model kept the tables alive for some time, but the main developers, influenced by the success of Ajax implementations and a huge number of new interactive apps gradually went back to div tags and CSS styles into a standard. Finally the Traditional Box Model was adopted on a large scale.

- universal selector :
Let's start with some basic rules that will help us provide consistency to the design:

* { margin: 0px; padding: 0px; }
Listing 2-26. General CSS rule.

Normally, for most elements, we need to customize the margins or just keep them to a minimum. Some default elements have margins that are non-zero and in most cases too wide. As we go through the creation of our design we will find that most of the elements used must have a margin of 0 pixels. To avoid having to constantly repeat styles, we can use the universal selector.

The first rule in our CSS file, presented in Listing 2-26, ensures that that every element will have an inner and outer margin of 0 pixels. From now on we will only need to modify the margins of the elements that we want to be greater than zero.

Basic concepts :
Remember that in HTML each element is considered as a box. the margin is actually the space around the element, the one outside the border of that box (the padding style, on the other hand, it is the space around the content of the element but inside its borders, such as the space between the title and the border of the virtual box formed by the h1 element that contains that title).

Margin size can be defined by specific sides of the element or all its sides at once. The style margin: 0px in our example sets a 0 or null margin for each element of the box. If the size had been specified as 5 pixels, for example, the box would have a space 5 pixels wide all the way around. This means that the box would be separated from its neighbors by 5 pixels. We will return to this topic later in this chapter.
