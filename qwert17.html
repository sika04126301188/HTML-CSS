- transitions :
From now on, beautiful effects using dynamic transformations are accessible and easy to implement. However, a real animation requires a process of more than two steps. The transition property was included to smooth the changes, magically creating the rest of the steps that are implicit in the movement. Only by adding this property we force the browser r to take action, create for us all those invisible steps, and generate a smooth transition from one state to another.


Listing 3-26. A beautiful rotation using transitions.

As you can see in Listing 3-26, the transition property can take up to four parameters separated by a space. The first value is the property that will be considered to make the transition (in our example we chose transform). This is necessary because several properties can change at the same time and we will probably need to create the process steps of transition only for one of them.

The second parameter specifies the time that the transition will take to go from the position start to finish. The third parameter can be any of the following keywords: ease, linear, ease-in, ease-out or ease-in-out. These keywords determine how the transition process will be performed based on a Bézier curve. Each of them represents different types of Bézier curve, and the best way to know how they work is watching them work on screen. The last parameter for the transition property is the delay. This indicates how long it will take for the transition on begin.

To produce a transition for all properties that are changing in an object, the keyword all must be specified. We can also declare multiple properties at once by listing them separated by commas.
