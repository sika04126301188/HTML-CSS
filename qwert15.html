_ HSLA :

Just as the rgba() function adds an opacity value to rgb(), the hsla() function does the same thing for the hsl() function. The hsla() function is just a different function for generating colors, but it's more intuitive than rgba(). some designers you'll find it easier to generate a custom color set using hsla(). The syntax of this function is: hsla(hue, saturation, luminosity, opacity).

#title { font: bold 36px MyNewFont, verdana, sans-serif;
text-shadow: rgba(0,0,0,0.5) 3px 3px 5px; color: hsla(120, 100%, 50%, 0.5); }
Listing 3-16. New color for title using hsla().

Following the syntax, hue represents the color extracted from an imaginary wheel and is expressed in degrees from 0 to 360. Close to 0 and 360 are the red colors, about 120 the green ones and about 240 the blue ones. The saturation value is represented as a percentage, from 0% (grayscale) to 100% (full color or fully saturated). Lightness is also a percentage value from 0% (completely dark) to 100% (completely lit). The 50% value represents normal or average brightness. the last value, just like in rgba(), it represents the opacity.

- Outline :
The outline property is an old CSS property that has been expanded in CSS3 to include an offset value. This property was used to create a second border, and now that border can be displayed away from the element's actual border.

#main { display: block; width: 500px; margin: 50px auto;
padding: 15px; text-align: center; border: 1px solid #999999;
background: #DDDDDD; outline: 2px dashed #000099; outline-offset: 15px; }
Listing 3-17. Adding a second border to the header.

In Listing 3-17 we add a second 2-pixel border to the styles originally applied to our template box. with a displacement of 15 pixels. The outline property has similar characteristics and uses the same parameters as border. The outline-offset property only needs a value in pixels.

- Border-image :
The possible effects achieved by the border and outline properties are limited to simple lines and only a few configuration options. The new border-image property was incorporated to overcome these limitations and leave the quality and variety of images up to the designer. borders available offering the alternative of using own images.

Follow the next link to download the diamonds.png file from our website and then copy this file to the same directory (folder) where it is find its CSS file: www.minkbooks.com/ content/diamonds.png. The border-image property takes an image and uses it as a pattern. According to the given values, the image is cut like a cake, the obtained parts are then placed around the object to build the border.

To get the job done, we need to specify three attributes: the name of the image file, the size of the pieces we want get from the pattern and some keywords to declare how the pieces will be distributed around the object.

Listing 3-18. A custom border for the header.

With the modifications made in Listing 3-18 we are defining a border of 29 pixels for the our header box and then loading the diamonds.png image to build that border. The value 29 in the border-image property declares the size of the pieces and stretch is one of the available methods to distribute these pieces around the box. There are three possible values ​​for the last attribute. The repeat keyword will repeat the pieces taken from the image as many times as necessary to cover the side of the element.

In this case, the size of the pieces is preserved and the image will be cut if there is no more space to place it. The round keyword will consider how long the side to be covered is and adjust the size of the pieces to make sure they cover the entire side and no pieces are cut off. Finally, the keyword stretch (used in the Listing 3-18) stretch just one piece to cover the entire side. In our example we use the border property to define the border size, but you can also use border-with to specify different sizes for each side of the element (The border-with property uses four parameters, with syntax similar to margin and padding.) The same goes for size. of each piece, up to four values ​​can be declared to get different images of different sizes from the pattern.
