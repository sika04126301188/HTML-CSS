_ linear gradient :

Gradients are one of the most attractive effects among those built into CSS3. This effect was practically impossible to implement using previous techniques but now it's really easy to do using CSS. A background property with some few parameters is enough to turn your document into a professional looking web page:


Listing 3-11. Adding a beautiful background gradient to our box.

Gradients are set as backgrounds, so we can use the background or background-image properties to declare them. The syntax for values ​​declared in these properties is linear-gradient(start position, start color, end color). The attributes of the linear-gradient() function indicate the starting point and the colors used to create the gradient. The first value can be specified in pixels, percentage or using the keywords top, bottom, left and right (as we did in our example). The start point can be replaced by an angle to declare a specific direction of the gradient:

background: linear-gradient(30deg, #FFFFFF, #006699);
Listing 3-12. Gradient with a direction angle of 30 degrees.

We can also declare the termination points for each color: background: linear-gradient(top, #FFFFFF 50%, #006699 90%));
Listing 3-13. Declaring termination points.

-radial gradient :
The standard syntax for radial gradients only differs in a few ways from the previous one. We must use the radial-gradient() function and a new attribute for the shape:

background: radial-gradient(center, circle, #FFFFFF 0%, #006699 200%);
Listing 3-14. radial gradient.

The start position is the origin and can be declared in pixels, percentage or a combination of the keywords center, top, bottom, left and right. There are two possible values ​​for the shape (circle and ellipse) and the ending for the color indicates the color and the position where the transitions start.

IMPORTANT :
At this time the gradient effect has been implemented by browsers in different ways . What we have learned in this chapter is the standard proposed by W3C (World Wide Web Consortium). Browsers like Firefox and Google Chrome already incorporate an implementation that works with this standard, but Internet Explorer and others are still busy On it. As always, test your codes in every browser available on the market to check the current state of the different implementations.

- RGBA :
Until now colors were declared as solid using hexadecimal values ​​or the rgb() function for decimals. CSS3 has added a new function called rgba() that simplifies color and transparency mapping. This function also solves a problem previous triggered by the opacity property. The rgba() function has four attributes. The first three are similar to those used in rgb() and simply declare the values ​​for the colors red, green, and blue in decimal numbers from 0 to 255. The last one, in change, corresponds to the new opacity ability. This value must be within a range from 0 to 1, with 0 being fully transparent and 1 as fully opaque.

#title { font: bold 36px MyNewFont, verdana, sans-serif;
text-shadow: rgba(0,0,0,0.5) 3px 3px 5px; }
Listing 3-15. Enhancing the shadow of the text with transparency.

Listing 3-15 offers a simple example that demonstrates how effects are enhanced by applying transparency. We replace the function rgb() by rgba() in shadow of the title and add an opacity/transparency value of 0.5. Now the shadow of our title will blend with the background, creating a much more natural effect. In previous versions of CSS we had to use different techniques in different browsers to do a transparent element. They all had the same problem: an element's opacity value was inherited by its children. That problem was solved by rgba() and we can now assign an opacity value to the background of a box without affecting its content.
