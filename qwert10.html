_ Box-sizing :

There is an additional property built into CSS3 related to the structure and the Traditional Box Model. The box-sizing property allows us to change how the total space occupied by an element on the screen will be calculated forcing browsers to include the values ​​of the padding and border properties in the original width. What explained above, each time the total area occupied by an element is calculated, the browser gets the final value using the following formula: size + margins + internal margins + borders. For this reason, if we declare the width property equal to 100 pixels, margin equal to 20 pixels, padding equal to 10 pixels and border equal to 1 pixel, the total horizontal area occupied by the element will be: 100+40+20+2= 162 pixels (note that we had to double the margin, padding and border values ​​in the formula because we consider that they were assigned both for both the right and left sides). This means that whenever you declare the width of an element with the property width, you will need to remember that the actual area to place the element on the screen will most likely be larger. Depending of your habits, it might sometimes be useful to force the browser to include the padding and border values ​​in the size of the element. In this case the new formula would be simply: size + margins.

div {
width: 100px;
margin: 20px;
padding: 10px;
border: 1px solid #000000;
-moz-box-sizing: border-box;
-webkit-box-sizing: border-box;
box-sizing: border-box; }
Listing 2-36. Including padding and border in the size of the element.

The box-sizing property can take two values. By default it is configured as content-box, which means that browsers will add the padding and border values ​​to the size specified by width and height. Wearing the border-box value instead, this behavior is changed so that padding and border are included inside of the element. Listing 2-36 shows the application of this property on a div element. This is just an example and we're not going to use it in our template, but it can be useful for some designers depending on how familiar are with the traditional methods proposed by previous versions of CSS.

IMPORTANT :
At this point, the box-sizing property, like other important CSS3 properties studied in upcoming chapters, it is experimental in some browsers. To effectively apply your documents, you must declare it with the appropriate prefixes, as we did in Listing 2-36. prefixes for the most common browsers are the following:

-*- -moz- for Firefox.
-*- -webkit- for Safari and Chrome.
-*- -o- for Opera.
-*- -khtml- for Konqueror.
-*- -ms- for Internet Explorer.
-*- -chrome- specific to Google Chrome.

- quick reference :
In HTML5 the responsibility for the presentation of the structure on the screen is more than ever in the hands of CSS. additions and Enhancements have been made in the latest version of CSS to provide better ways to organize documents and work with their elements.

Attribute selector and pseudo classes

CSS3 incorporates new mechanisms for referencing HTML elements.

- Attribute Selector :
Now we can use attributes other than id and class to find elements in the document and assign styles. With the keyword[attribute=value] construct, we can reference an element that has a particular attribute with a specific value. For example, p[name=”text”] will reference each p element with an attribute called name and the value "text". CSS3 also provides techniques to make this reference even more specific. Using the following combinations From symbols ^=, $= and *= we can find elements that start with the supplied value, elements that end with that value and elements that have the provided text somewhere in the attribute value. For example, p[name^=”text”] will be used to find p-elements that have an attribute called name with a value that starts with ”text”.

Pseudo Class :
nth-child() This pseudo class finds a specific child by following the HTML tree structure. For example, with the style span:nth-child(2) we are referencing the span element that has other span elements as siblings. and is located at position 2. This number is considered the index. Instead of a number we can use the keywords odd and even to reference elements with an odd or even index respectively (for example, span:nth child(odd)).

Pseudo Class :
first-child This pseudo class is used to reference the first child, similar to :nth-child(1).

Pseudo Class :
last-child This pseudo class is used to reference the last child.

Pseudo Class :
only-child This pseudo class is used to reference an element that is the only child. available from the same parent element.

Pseudo Class :
not() This pseudo class is used to reference every element except the declared one. in parentheses.

- Selectors :
CSS3 also introduces new selectors that help get to elements that are hard to reference using other techniques. Selector This selector references the element on the right when it has the element on the left as its parent. For example, div p will reference each element p that is a child of a div element.

- Selector :
+ This selector references elements that are siblings. The reference will point to the element on the right when it is immediately preceded by the one on the left. For example, span + p will affect p elements that are siblings and are located after a span element.

Selector ~ This selector is similar to the previous one, but in this case the element on the right does not have to be located immediately after the one on the left.
